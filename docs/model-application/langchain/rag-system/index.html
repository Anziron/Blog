RAG 系统构建

RAG (Retrieval-Augmented Generation) 是一种结合了检索系统和生成式AI的技术，它允许语言模型访问和引用外部知识，从而生成更准确、更丰富的回答。

RAG的基本原理​

RAG系统的工作流程通常包括以下步骤：

  1. 索引: 将文档转换为向量形式并存储
  2. 查询: 将用户问题转换为向量并查找相关文档
  3. 增强: 将检索到的信息添加到提示中
  4. 生成: 利用增强后的提示让LLM生成回答

使用LangChain构建RAG系统​

1. 文档加载与分割​

from langchain.document_loaders import PyPDFLoader, DirectoryLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 加载文档
loader = DirectoryLoader('./documents/', glob="**/*.pdf", loader_cls=PyPDFLoader)
documents = loader.load()

# 文档分割
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = text_splitter.split_documents(documents)

2. 创建向量数据库​

from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma

# 初始化嵌入模型
embeddings = OpenAIEmbeddings()

# 创建向量数据库
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

3. 创建检索器和问答链​

from langchain.chains import RetrievalQA
from langchain.llms import OpenAI

# 创建检索器
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 5}
)

# 创建问答链
qa_chain = RetrievalQA.from_chain_type(
    llm=OpenAI(),
    chain_type="stuff",
    retriever=retriever,
    return_source_documents=True
)

4. 查询RAG系统​

# 查询系统
result = qa_chain({"query": "什么是机器学习？"})

# 打印回答
print(result["result"])

# 打印来源文档
for doc in result["source_documents"]:
    print(f"来源: {doc.metadata}")
    print(f"内容: {doc.page_content[:100]}...")

RAG系统的优化​

  • 嵌入模型选择: 不同的嵌入模型对检索质量有显著影响
  • 检索策略: 尝试不同的检索方法(相似度搜索、混合搜索等)
  • 重新排序: 利用交叉编码器等技术对检索结果重新排序
  • 提示工程: 优化与检索结果结合的提示模板

RAG系统是当前LLM应用中最实用的技术之一，能够有效解决幻觉问题并提供基于事实的回答。
